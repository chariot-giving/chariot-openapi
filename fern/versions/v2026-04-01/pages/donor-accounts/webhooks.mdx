Subscribe to these events via an [Event Subscription](/api/event-subscriptions) to drive your internal workflows. All events include an `associated_object_id` you can use to fetch the full object.

## Account Events

| Event | When it fires |
|---|---|
| `donor_account.created` | A new Donor Account has been created. In the Donor-Initiated Verification flow, this is your signal that a donor has submitted a grant and is waiting for verification. |
| `donor_account.updated` | The Donor Account was updated — including when it transitions to `approved` or `rejected`. Fetch the Donor Account to get its current status. |
| `donor_account.rejected` | The Donor Account transitioned to `rejected`. All open Grant Requests tied to it are automatically canceled. |

## Grant Request Events

| Event | When it fires |
|---|---|
| `grant.created` | A new Grant Request was submitted via DAFpay. May arrive before `donor_account.created` if both are created in the same flow. |
| `grant.updated` | A Grant Request was updated — including when it transitions to `initiated` or `canceled`. Fetch the Grant Request to get its current status. |

## Setting up an Event Subscription

<CodeBlocks>
```bash cURL
curl -X POST https://api.givechariot.com/v1/event_subscriptions \
  -H "Authorization: Bearer $CHARIOT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-server.example.com/webhook",
    "enabled_events": [
      "donor_account.created",
      "donor_account.updated",
      "donor_account.rejected",
      "grant.created",
      "grant.updated"
    ]
  }'
```
```typescript TypeScript
await chariot.eventSubscriptions.create({
  url: "https://your-server.example.com/webhook",
  enabled_events: [
    "donor_account.created",
    "donor_account.updated",
    "donor_account.rejected",
    "grant.created",
    "grant.updated",
  ],
});
```
</CodeBlocks>
